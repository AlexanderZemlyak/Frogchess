\# Frog Chess



Веб-игра Frog Chess с клиентом и сервером.



\## Технологии



\- \*\*Бэкенд:\*\* ASP.NET

\- \*\*Фронтенд:\*\* Phaser



\## Запуск



```bash

dotnet run

```



\## Архитектурное решение



Выбран гибридный подход с использованием REST для реализации операций, которые меняют состояние игры и SignalR для рассылки уведомлений и отслеживания отключений.



Плюсы подхода:
1) Для реализации операций получаем возможность использовать http семантику, то есть стандартный набор методов (применяется POST и PATCH) и стандартные коды ошибок.

2\) Упрощение тестирования операций за счет того, что не надо явно организовывать WebSocket-соединения.

3\) Разделение ответственности: бизнес-логика в контроллерах и сервисах, хаб только для уведомлений.



Ускорение сетевого взаимодействия, которое достигается при использовании чистого SignalR подхода не критично, поскольку это пошаговая игра.
